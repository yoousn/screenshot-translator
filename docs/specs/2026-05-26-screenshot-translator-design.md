> **创建时间**：2026-05-26  
> **设计人**：Antigravity & 阁下  
> **部署宿主机**：N100 家庭服务器 (192.168.1.3 / *.yousn.me)

## 1. 整体架构设计

本软件采用 **“极致轻量化 C++ 客户端 + N100 集中式处理服务端”** 的 SaaS 架构模式。所有复杂的 AI 推理（OCR）、网络翻译、以及图形像素处理（擦除、重绘）全部在 N100 服务器端运行，客户端只负责系统截图热键、框选交互、配置界面与图片显示。

### 1.1 系统架构图

```mermaid
graph TD
    subgraph Client [用户本地电脑 (Windows)]
        QtApp[C++/Qt 截图客户端]
        ConfigUI[配置面板 UI]
    end

    subgraph N100 [N100 宿主机 (飞牛 OS / 192.168.1.3)]
        OpenResty[OpenResty 反向代理 80/443]
        PyServer[Python FastAPI 服务端]
        LocalNewAPI[new-api 中转服务 Port 3001]
        PaddleOCREngine[PaddleOCR 引擎 (CPU)]
        ImageProcessor[图像处理器 (OpenCV + Pillow)]
    end

    QtApp <-->|HTTPS + API Key| OpenResty
    OpenResty <-->|本地环回| PyServer
    PyServer <-->|内网 http://192.168.1.3:3001| LocalNewAPI
    PyServer -->|调用| PaddleOCREngine
    PyServer -->|调用| ImageProcessor
    PyServer <-->|公网 HTTP| ExternalAPIs[外网翻译 API: 百度/谷歌网页版等]
```

### 1.2 架构设计决策

* **C++ 客户端尺寸**：保持在约 10MB ~ 20MB 左右，极速秒开，无需在本地安装 CUDA、Python、深度学习框架等大文件。
* **高内聚低耦合**：C++ 端无需理解百度的签名算法、大模型的 API 请求格式或图像擦除填充逻辑。所有的 API 管理、调用逻辑、签名计算和图形处理都在 N100 上的 Python 中处理，极易扩展。
* **内外网双通机制**：
  * **内网模式**：直连宿主机 `http://192.168.1.3:8318`，速度最快（延迟 <5ms）。
  * **公网模式**：通过 `https://ocr.yousn.me` 经过 OpenResty 反向代理并使用 frpc 穿透出站，并在请求头中携带 `X-API-Key` 鉴权安全验证。
* **安全鉴权**：N100 端设置一个自定义的 `Client-Token`（在 Python 配置文件中配置，并在 C++ 中输入）。任何没有正确 Token 的请求一律拦截，防止 N100 上的 API 被非法盗刷。

## 2. 客户端 (C++/Qt) UI 与交互设计

客户端核心采用 Qt 6 (或 Qt 5) 开发，主要分为**截图交互层**和**配置面板层**。

### 2.1 截图交互工作流

1. **热键触发**：全局监听 `Alt + A`（正常截图）和 `Alt + T`（截图并翻译）。
2. **框选蒙版**：屏幕变暗，用户按下鼠标左键拖动框选区域。
3. **悬浮工具栏**：松开鼠标后，框选区域下方显示悬浮工具栏。
   * **功能按钮**：复制（Ctrl+C）、保存（Ctrl+S）、钉图（Pin）、**翻译（Ctrl+Q）**。
4. **加载中状态**：如果触发翻译（按 Ctrl+Q 或点击翻译按钮），框选区域上浮现旋转加载动画（Loader），并阻断多余点击。
5. **替换显示**：收到服务器返回的处理后图像数据，将框选区原图原地替换显示为翻译图。此时用户点击“复制”或“保存”得到的就是已翻译并嵌好字的图像。

### 2.2 配置面板 UI 设计

* **“获取模型”按钮交互**：
  * 点击后，客户端把当前的 `中转地址` 和 `API Key` 作为临时参数发送给 N100 后端接口。
  * N100 联网调用该 `new-api` 的 `/v1/models`，获取所有大模型 ID 列表，并返回给客户端。
  * 客户端收到后，动态更新`模型名称`下拉框（ComboBox），供用户方便地选择（如 `gemini-1.5-flash`, `gpt-4o-mini` 等）。
* **“点击验证”按钮交互**：
  * 客户端把当前配置的所有翻译配置打包，POST 发送给 N100 的 `/api/config/test` 接口。
  * 如果接口返回成功，右侧显示绿色的 `验证成功` 并回显测试翻译出的汉字；如果失败，则红字回显详细报错原因。
  * 验证成功后，N100 自动在服务器端写入此配置并持久化，客户端保存设置并退出配置面板。

## 3. 服务端 (N100 / Python) 图像与翻译处理设计

N100 服务端采用 **FastAPI** 框架开发，提供极佳的异步性能和简洁的路由编写。

### 3.1 OCR 文字识别管道 (PaddleOCR)

* **模型加载**：FastAPI 启动时一次性实例化 `PaddleOCR(use_angle_cls=True, lang="ch")`，常驻内存，后续请求直接热重用。
* **输入处理**：识别图像中的所有文字行，获取每个文字行的多边形边界框（Bounding Box, 通常由左上、右上、右下、左下 4 个二维点坐标决定）和识别文本。

### 3.2 图像像素处理管道 (OpenCV + Pillow)

对于每一个识别出的文字框：
1. **背景颜色采样（像素分析）**：
   * 在框的四周建立一个外扩 2 像素的“环形边缘采样区”（Border Ring）。
   * 排除文字边缘像素，提取该采样区内像素的**中位数颜色**（Median Color）作为该文字区域的基准背景色 $C_{bg}$。
2. **文字区域擦除（像素涂改）**：
   * 使用 OpenCV 的 `cv.fillPoly` 算法，将文字对应的多边形区域填充为采样出的背景色 $C_{bg}$，达到无痕抹除原文字的目的。
3. **前景色（字体颜色）采样**：
   * 在涂抹前，采样文字框内部的像素，排除背景色，提取出极值色（如深色文字提取最暗色，浅色文字提取最亮色），作为译文重绘时的字体前景色 $C_{fg}$。
4. **译文排版与绘制（Pillow / FreeType）**：
   * 计算原多边形框的宽度 $W_{box}$ 和高度 $H_{box}$。
   * 计算译文长度。如果译文在当前默认字号下宽度超过 $W_{box}$，自动进行折行（Wrap）。
   * 如果折行后高度依然超标，则动态等比缩小字体号，直到可以完美装入原文字框中。
   * 使用 Pillow 库的 `ImageDraw.text`，采用前景色 $C_{fg}$，将翻译后的中文居中绘制在擦除后的纯色区域内。

## 4. 服务端 API 接口定义 (RESTful JSON)

服务端绑定在 N100 上的端口，默认启动 `8318` 端口（与 Hint 一致）。

### 4.1 全局鉴权

所有请求的 HTTP Header 必须携带：
`X-API-Key: <用户的 N100 服务鉴权Token>`

---

### 4.2 截图翻译接口：`POST /api/translate`

* **请求格式**：`multipart/form-data`
* **参数**：
  * `image`: 截图的二进制文件 (PNG/JPG)。
* **返回格式**：直接返回处理后的二进制图片数据 (`image/png`)。
* **扩展 Headers**（可选）：
  * `X-Translation-Status`: `Success`
  * `X-OCR-Count`: 识别出的文本行数。

---

### 4.3 测试配置与验证接口：`POST /api/config/test`

* **请求格式**：`application/json`
* **请求 Payload**：
  ```json
  {
    "channel": "new-api", // "new-api" | "baidu" | "google"
    "config": {
      "base_url": "http://192.168.1.3:3001", // 可空，支持域名 api.yousn.me
      "api_key": "sk-...",                   // 可空
      "model": "gemini-1.5-flash",           // 可空
      "app_id": "",                          // 百度 AppId
      "secret_key": ""                       // 百度密钥
    }
  }
  ```
* **逻辑**：
  * 服务端临时实例化对应通道的翻译器，向外网服务发起 `"Test Connection"` 的翻译请求。
  * 如果成功翻译为 `"测试连接"`，服务端将此配置安全写入 N100 上的本地 `config.yaml` 配置文件中。
* **返回格式**：`application/json`
  * 成功：
    ```json
    {
      "status": "success",
      "result": "测试连接"
    }
    ```
  * 失败：
    ```json
    {
      "status": "failed",
      "error": "请求超时或 API Key 无效: 详细错误堆栈"
    }
    ```

---

### 4.4 动态模型获取接口：`POST /api/config/fetch_models`

* **请求格式**：`application/json`
* **请求 Payload**：
  ```json
  {
    "base_url": "api.yousn.me", // 或内网地址 http://192.168.1.3:3001
    "api_key": "sk-..."
  }
  ```
* **逻辑**：
  * N100 服务端拼接 `base_url` 与 `/v1/models`，发起 `GET` 请求。
  * 解析返回的 OpenAI 标准模型格式，提取出 `data` 数组中所有对象的 `id` 属性。
* **返回格式**：`application/json`
  * 成功：
    ```json
    {
      "status": "success",
      "models": ["gemini-1.5-flash", "gpt-4o-mini", "claude-3-5-sonnet"]
    }
    ```
  * 失败：
    ```json
    {
      "status": "failed",
      "error": "无法连接中转地址，请检查网络或密钥"
    }
    ```
